ТЕХНИЧЕСКОЕ ЗАДАНИЕ: Панель Супер-Админа (Admin Panel)

Проект: ClassPlay (SaaS)

1. Введение

Панель Супер-Админа — это закрытая часть системы для управления пользователями и мониторинга. Доступ к ней имеют только владельцы продукта.
Стиль: Темный "Enterprise" дизайн (Dark Sidebar + Light Content).

2. Дизайн и Навигация

Стилистика:

Сайдбар: Темно-синий (#0F172A).

Контентная часть: Белая/Серая.

Таблицы: Строгий стиль, зебра-строки.

Структура Меню (Сайдбар):

Обзор (Overview): Дашборд со статистикой.

Ученики (Students): (Опционально, если админ управляет и учениками).

Учителя (Teachers): Основной раздел управления клиентами.

Настройки (Settings): Глобальные переменные.

3. Функциональные Модули

3.1. Управление Учителями (Teachers Management)

Таблица:

Колонки: Имя, Email (Логин), Школа, Дата окончания подписки, Статус.

Действия (CRUD):

Создать: Кнопка "Добавить учителя". Модальное окно с полями (ФИО, Email, Пароль, Срок лицензии).

Редактировать: Изменение пароля или продление подписки.

Блокировать: Смена статуса на "Inactive" (запрет входа).

Удалить: Полное удаление аккаунта.

3.2. Авторизация Админа

Вход: Отдельная ссылка на экране логина ("Вход для администратора").

Безопасность:

Проверка роли role: 'admin' в базе данных.

При попытке зайти на /admin без прав — редирект на логин.

3.3. Мониторинг (Overview)

Виджеты (KPI):

Всего пользователей: Общее число зарегистрированных учителей.

Активных сегодня: Уникальные входы за 24 часа.

Генераций AI: Общее количество запросов к API (для контроля расходов).

4. Технические Требования

База данных: Firebase Firestore / PostgreSQL.

Права доступа: Строгая проверка токена на бэкенде при любом запросе к /api/admin/*.
